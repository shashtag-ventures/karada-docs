Projects are the core organizational unit in Karada. Every deployment, integration, and usage metric is tied to a specific project.

## Creating a Project

You can create two types of projects:

1. **GitHub Repository**: Connect a GitHub repo to automatically deploy infrastructure.
2. **Provider**: Set up an integration-first project, such as an Auto-MCP server.

## Project Dashboard

The Project Dashboard provides a centralized view of your project's health:

- **Overview**: See active deployments and connected repositories.
- **Usage**: Monitor the number of requests, compute time, and resources consumed by your project.
- **Settings**: Configure environment variables, rename your project, or delete it permanently.

## Scope Switcher

If you are a member of multiple teams, you can quickly switch between them using the Team Switcher in the top navigation bar. Every project belongs to exactly one team.
