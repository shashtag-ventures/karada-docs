Deploying your code should be invisible. Karada handles the heavy lifting of taking your code from a GitHub repository to a live, scalable environment.

## Triggering a Deployment

Deployments are triggered automatically whenever you push to your configured production branch (usually `main` or `master`).

You can also trigger manual deployments from the Project Dashboard by clicking **Deploy**.

## Deployment Logs

During a deployment, Karada streams real-time build logs directly to your dashboard. If a build fails, you can inspect the exact error and environment variables that caused the issue.
